# Git Workflow

---

## Типы веток
- feature/ - пример: новая фича
- fix/ - пример: починка бага
- docs/ - пример: изменения project-structure.md
- chore/ - пример: настройка среды

---

## Конвенция именования коммитов
Сообщение в коммите должно описывать что именно сделано (желательно на английском)
Нельзя:
- просто буквы или цифры
- init, commit, fix итд
Примеры:
- fixed api/users
- implemented parsing
- documented jwt chains

---

## Процесс работы:
1. Создаем issue
2. В issue делаем create branch
3. Открываем branch в github desktop (оно делает checkout и переносит незакомиченный код на эту ветку)
4. Пишем код или доки
5. Пушим в эту ветку
6. Делаем pr запрос
7. Проверяем очень внимательно, чтоб было быстрее вы сами можете мерджить
8. Мерджим в мейн
9. Повторяем с первого пункта
(не считая кода занимает **2-3** минуты)

---

## Процесс коммуникации
Чтобы запросить что-то (фичу, починку бага) переходим в нужный репозиторий, открываем *issue*, сообщаем ответсвенному в чате

## Решение проблем
Если что-то пошло не так или кажется что что-то может пойти не так, **не чиним сами**, пишем адниму
